# 知识库文档

## 功能说明

分析 `/home/jonson/cuda-learning/` 目录下的内容，汇总知识库文档存放到当前目录，按照知识类别存放到对应的目录。

## 作用

作为知识库提供给大模型（豆包、通义等），让大模型作为面试官进行面试提问。知识库文档应该包含详细的技术细节、代码示例和知识点说明，引导面试者深入熟悉和理解各个知识点。

## 任务要求

### 文件命名规则

- **按照知识类型命名**，而不是统一命名为"知识库.md"
- 命名格式：`{知识类型}知识库.md`
- 示例：
  - `C++知识库.md`
  - `CUDA知识库.md`
  - `TSE知识库.md`
  - `项目实践知识库.md`
  - `JD分析知识库.md`
  - `学习计划知识库.md`

### 内容要求

1. **内容来源要求（重要）**
   - **必须基于当前工程中的实际学习材料**：知识库的所有内容都应该来源于工程目录下的实际文件，如学习讲义、训练题目、实践代码、技术文档等
   - **禁止使用通用理论知识**：不要使用网络上通用的理论知识，而是要基于工程中实际存在的学习内容进行汇总
   - **先分析后汇总**：在创建知识库文档前，必须先详细分析工程目录结构，查看实际的学习材料文件，了解工程中具体包含哪些知识点
   - **反映实际学习进度**：知识库应该反映工程中的实际学习状态和内容，包括已完成的学习阶段、重点知识点、实践项目等

2. **内容要详细且有细节**
   - **详细阐述知识点**：每个知识点都要有充分的文字说明，包含概念、原理、应用场景、实践经验等维度
   - **深入解释技术细节**：不仅要说明"是什么"，还要说明"为什么"和"怎么做"，包括实现原理、底层机制、使用场景等
   - **包含代码示例**：适当地包含代码示例有助于理解和学习，可以从工程中的实践代码提取关键代码片段
   - **引导面试者熟悉知识点**：通过详细的解释、代码示例、常见问题等，帮助面试者深入理解和掌握知识点

3. **代码示例的使用**
   - **适量使用代码**：可以包含代码示例来辅助说明技术要点，但要适量，避免过多代码堆砌
   - **关键代码片段**：优先展示关键代码片段，突出重点，可以通过注释说明关键点
   - **代码来源**：代码应该来自工程中的实际实践代码，确保准确性和实用性
   - **配合文字说明**：代码示例要有充分的文字说明，解释代码的作用、原理和使用场景

4. **内容要详细且准确**
   - 每个知识点都要有充分的文字说明和细节描述
   - 包含概念、原理、应用、实践经验、常见问题等维度
   - 为面试提问提供丰富的知识素材，引导面试者熟悉和理解
   - 确保内容与工程中的实际学习材料保持一致

### 工程说明要求

- **不需要过于详细的工程说明**
- **必须深入分析当前工程状态**：工程会持续更新，每次创建知识库文档前，必须详细分析当前工程的实际结构和内容，具体包括：
  - 查看各学习目录下的实际文件（如README.md、学习讲义、训练题目、实践代码等）
  - 了解工程中包含的具体学习材料和知识点
  - 确认已完成的学习阶段和正在进行的内容
  - 查看项目实践目录中的实际题目和完成情况
  - 提取实践代码中的关键代码片段和技术要点
- **保持说明简洁**：只列出主要目录和内容分类，不需要深入到具体文件
- **重点放在知识内容的汇总和详细描述上**：知识库文档的核心是基于工程实际内容的知识汇总，包含详细的文字说明、代码示例和技术细节，引导面试者深入理解知识点
- **内容必须与工程同步**：知识库应该反映工程当前的实际状态，不能脱离工程内容编写通用知识

## 知识库汇总原则

### 汇总的目的

知识库文档的目的是**汇总**工程中的学习材料，而不是简单总结。汇总的特点：

1. **内容丰富详细**：包含详细的技术细节、实现原理、使用场景等
2. **包含代码示例**：适当地展示关键代码片段，帮助理解和学习
3. **引导深入理解**：通过详细的解释、示例、常见问题等，引导面试者深入熟悉知识点
4. **基于实际材料**：所有内容都来源于工程中的实际学习材料，确保准确性和实用性

### 汇总的方法

1. **提取关键知识点**：从学习讲义、训练题目、实践代码中提取关键知识点
2. **整理详细说明**：对每个知识点进行详细说明，包括概念、原理、应用等
3. **展示代码示例**：从实践代码中提取关键代码片段，配合文字说明
4. **组织知识结构**：按照逻辑顺序组织知识点，形成完整的知识体系

### 汇总的质量要求

1. **准确性**：内容必须与工程中的实际材料保持一致
2. **详细性**：包含足够的技术细节，能够深入理解知识点
3. **实用性**：代码示例和技术说明要实用，能够指导实际开发
4. **引导性**：能够引导面试者深入学习和理解知识点
