# 测试设计能力

## 能力清单

### 测试分析能力

#### 1级（1分）
1. 掌握KYM和TCO测试分析方法
2. 能对简单需求进行KYM和TCO分析
3. 会使用PPDCS进行建模

#### 2级（2分）
1. 使用KYM对复杂需求进行信息搜集，准确识别价值，信息全面、深入
2. TCO整理逻辑清晰、功能完备
3. 能够对复杂对象选择合适的建模方法进行建模

#### 3级（3分）
1. KYM对跨领域复杂对象进行信息搜集，准确识别价值，信息全面、深入
2. TCO整理逻辑清晰、功能完备
3. 能够对复杂对象使用多种建模方式，建模方法完备能覆盖被测对象

### 测试设计能力

#### 1级（1分）
1. 能够应用ATDD（验收测试驱动开发）、BDD（行为驱动开发）等测试设计方法输出测试用例清单，并归档RDC或者通过模型输出Tcon，并从Tcon编写测试用例
2. 主导实施特性测试分层实践
3. 可以分析测试用例是否可以自动化，给出分析结论
4. 能够设计合理的UT/FT/ST测试分层策略

#### 2级（2分）
1. 测试用例的编写使用边界值、等价类等基本方法，保证用例覆盖的完备性
2. 准确识别用例是否可自动化，只要可自动化都能自动化
3. 测试设计考虑威胁建模及安全风险分析，并输出相应的安全测试用例

#### 3级（3分）
可通过现场故障现象或复盘或现场KPI数据，提取或优化测试用例

### 测试策略制定能力

#### 1级（0.5分）
能够根据测试设计制定合适的、可执行的测试策略

#### 2级（1分）
制定高效、准确的测试策略，包括需求变更波及的范围（受影响的组件，接口，版本分支、其它项目）

#### 3级（1.5分）
主导实施特性测试分层实践

## 学习资源

### 📚 理论知识

#### 测试设计基础方法
1. **等价类划分法**
   - 概念：将输入数据分为有效和无效等价类
   - 应用：识别测试数据范围
   - 优点：减少测试用例数量，提高效率

2. **边界值分析法**
   - 概念：测试输入范围的边界值
   - 应用：测试最大值、最小值、边界外值
   - 优点：边界是最容易出错的地方

3. **场景法（流程测试）**
   - 概念：按照用户使用场景设计测试
   - 应用：测试业务流程、异常流程
   - 优点：贴近真实用户使用情况

4. **错误推测法**
   - 概念：基于经验推测可能出错的地方
   - 应用：补充遗漏的测试场景
   - 优点：利用经验发现潜在问题

#### 测试分析方法
- **KYM（Know Your Mission）**：了解你的使命，信息搜集方法，用于需求分析和价值识别
- **TCO（Test Coverage Outline / Testing Coverage Outline）**：测试覆盖大纲，整理测试覆盖范围
- **PPDCS建模**：测试对象建模方法，使用Process-Product-Data-Control-State维度描述测试对象
- **Tcon（Test Context）**：测试场景，从模型输出的测试场景

#### 术语说明
详见本目录或[学习总结](../学习总结/)中的完整术语表。

#### 测试驱动方法
- **ATDD（验收测试驱动开发）**：先写验收测试，再开发
- **BDD（行为驱动开发）**：用Given-When-Then描述行为
