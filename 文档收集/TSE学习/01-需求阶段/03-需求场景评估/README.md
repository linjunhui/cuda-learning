# 需求场景评估

## 能力要求

### 1级（0.5分）
- 能对需求描述的场景正确性进行验证

### 2级（1分）
- 能对需求场景的完备性进行验证，并提出BA/SE未考虑到的场景

### 3级
无

## 学习资源

### 📚 理论知识

#### 核心概念
1. **场景正确性验证**
   - 场景是否符合实际情况？
   - 场景是否符合业务规则？
   - 场景是否符合安全要求？
   - 场景是否可以实现？

2. **场景完备性验证**
   - 正常场景（主流程）
   - 异常场景（错误处理）
   - 边界场景（特殊情况）
   - 扩展场景（其他可能）

3. **场景识别方法**
   - 按流程检查
   - 按用户角色检查
   - 按环境检查
   - 错误推测

### 📖 推荐书籍

1. **《用例分析技术》** (Alistair Cockburn)
   - 场景分析和用例方法

2. **《需求工程》** (Karl Wiegers)
   - 需求场景分析方法
