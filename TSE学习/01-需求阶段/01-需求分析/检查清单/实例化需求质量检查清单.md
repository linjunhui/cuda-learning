# 实例化需求质量检查清单

## 使用说明

本检查清单专门用于评估实例化需求（Specification by Example）的质量。实例化需求的核心是用具体例子说明需求，确保团队对需求的理解一致。

---

## 1. 目标明确性检查

### 1.1 业务目标
- [ ] 业务目标清晰明确
- [ ] 目标可以追溯
- [ ] 目标与示例关联明确
- [ ] 目标价值可衡量

### 1.2 功能范围
- [ ] 功能范围从目标导出
- [ ] 包含的功能明确
- [ ] 不包含的功能明确
- [ ] 范围边界清晰

---

## 2. 示例质量检查

### 2.1 示例完整性
- [ ] 有具体的示例而非抽象描述
- [ ] 示例覆盖正常业务流程
- [ ] 示例覆盖异常流程
- [ ] 示例覆盖边界情况
- [ ] 示例数据真实有效

### 2.2 示例清晰性
- [ ] 示例描述清晰易懂
- [ ] 示例步骤明确
- [ ] 示例结果可验证
- [ ] 示例避免歧义

### 2.3 示例结构化
- [ ] 使用Given-When-Then格式
- [ ] 格式规范统一
- [ ] 结构清晰易读
- [ ] 示例之间逻辑清晰

---

## 3. Given（前提条件）检查

### 3.1 前提条件完整性
- [ ] 所有必要的前提条件已列出
- [ ] 前提条件具体明确
- [ ] 前提条件可重现
- [ ] 系统初始状态已描述

### 3.2 前提条件质量
- [ ] 避免模糊描述
- [ ] 使用具体数据
- [ ] 条件之间的关系明确
- [ ] 条件可验证

**示例对比**：
- ❌ 差：系统中有一些用户
- ✅ 好：系统中存在用户ID为"U001"，邮箱为"user@example.com"，状态为"激活"的用户

---

## 4. When（触发动作）检查

### 4.1 动作描述
- [ ] 触发动作明确具体
- [ ] 动作顺序清晰
- [ ] 用户操作步骤详细
- [ ] 系统响应动作已描述

### 4.2 动作完整性
- [ ] 包含所有关键操作步骤
- [ ] 操作参数明确
- [ ] 操作时机明确
- [ ] 操作上下文明确

**示例对比**：
- ❌ 差：用户提交表单
- ✅ 好：用户填写邮箱"user@example.com"和密码"Password123"，然后点击"登录"按钮

### 5.1 结果明确性
- [ ] 预期结果具体明确

### 5.2 结果完整性
- [ ] 包含所有重要结果
- [ ] 界面变化已描述

**示例对比**：
- ❌ 差：系统处理成功

### 7.1 正常流程

### 7.2 异常流程
- [ ] 错误情况已考虑
- [ ] 异常输入已覆盖
- [ ] 系统错误已处理
- [ ] 业务规则违反已考虑

### 7.3 边界情况
- [ ] 边界值已测试
- [ ] 极限情况已考虑

- [ ] 数据类型边界已覆盖

### 7.4 场景完整性评分
- **优秀**：覆盖所有重要场景（正常+异常+边界）
- **良好**：覆盖主要场景，少量边界情况缺失
- **一般**：仅覆盖正常流程
- **待改进**：场景覆盖不完整

## 8. 可执行性检查

### 8.1 可执行格式
- [ ] 使用可执行格式（如Gherkin）
- [ ] 格式符合工具要求

- [ ] 可以被BDD工具解析

### 8.2 可测试性
- [ ] 每个示例可独立测试
- [ ] 测试步骤可自动化
- [ ] 测试数据可准备
- [ ] 测试结果可验证

### 8.3 Gherkin格式示例

```gherkin
场景：用户成功登录
  给定 用户已注册且账号为"test@example.com"
  并且 用户密码为"Password123"
  并且 账号状态为"激活"
  当 用户输入邮箱"test@example.com"
  并且 用户输入密码"Password123"
  并且 用户点击"登录"按钮
  那么 系统显示"登录成功"消息
  并且 系统跳转到主页
  并且 系统显示欢迎消息"欢迎，test"
```

## 综合质量评分

### 评分维度

| 维度 | 权重 | 得分 | 加权得分 |
|------|------|------|---------|
| 示例质量 | 30% | ___ | ___ |
| 场景覆盖 | 25% | ___ | ___ |
| 可执行性 | 20% | ___ | ___ |
| 规则明确性 | 15% | ___ | ___ |
| 协作效果 | 10% | ___ | ___ |
| **总分** | 100% | - | **___** |

### 评分标准
- **优秀（90-100分）**：实例化需求质量很高，很好地体现了实例化需求的优点
- **良好（75-89分）**：质量良好，有少量改进空间
- **一般（60-74分）**：基本满足要求，需要改进
- **待改进（<60分）**：质量不足，需要重大改进

## 常见问题和改进建议

### 问题1：示例过于抽象
**问题**：使用抽象描述而非具体例子
**改进**：使用具体的输入数据和预期输出

### 问题2：场景覆盖不完整
**问题**：只包含正常流程
**改进**：补充异常流程和边界情况

### 问题3：前提条件不明确
**问题**：Given条件模糊
**改进**：使用具体数据和明确状态

### 问题4：结果不可验证
**问题**：Then结果无法验证
**改进**：使用可观察、可测量的结果
