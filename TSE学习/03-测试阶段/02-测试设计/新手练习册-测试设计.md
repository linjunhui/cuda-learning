## 练习1：计算器功能测试设计（最简单）

### 题目描述

设计一个简单"加法计算器"功能的测试用例。

**功能需求**：
- 用户可以输入两个数字
- 点击"="按钮计算两个数字的和
- 显示计算结果

### 你的任务

1. **使用等价类划分法**识别测试点
2. **使用边界值分析法**补充用例
3. **设计至少10个测试用例**

### 提示

考虑以下情况：
- 正常输入两个数字
- 输入非数字（字母、特殊字符）
- 输入为空
- 输入很大的数字
- 输入小数
- 输入负数

## 练习2：用户注册功能测试设计（基础）

### 题目描述

设计"用户注册"功能的测试用例。

**功能需求**：
- 用户可以注册新账号
- 需要输入：邮箱、密码、确认密码、手机号
- 邮箱格式：xxx@xxx.xxx
- 密码要求：6-20位，包含字母和数字
- 手机号：11位数字，以1开头

1. **使用等价类划分法**设计测试用例
2. **使用边界值分析法**测试密码长度

## 练习3：购物车功能测试设计（场景法）

### 题目描述

使用场景法设计"购物车"功能的测试用例。

**功能需求**：
- 用户可以添加商品到购物车
- 可以修改商品数量
- 可以删除商品
- 可以查看购物车总价

### 你的任务

1. **画出主流程**（正常场景）
2. **识别异常场景**
3. **设计每个场景的测试用例**

### 提示

**主流程**：
```
浏览商品 → 添加商品到购物车 → 查看购物车 → 修改数量 → 结算
```

**异常场景**：
- 商品库存不足
- 商品已下架
- 购物车为空时结算
- 数量超过库存
